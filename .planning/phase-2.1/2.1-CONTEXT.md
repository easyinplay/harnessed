## Implementation Decisions

### npx-skill-installer scope（gray area 1）
- **D-01:** npx-skill-installer 走 **minimal-pin 一个工具实装** — 选一个工具(research 指向 vercel-labs/skills 为 de-facto)minimal pin 实装,verify 走 best-effort。保证 6 install method 全覆盖。
- **D-02:** `~/.agents/` vs `~/.claude/skills/` 目录约定冲突 — 不在 Phase 2.1 引入 SessionStart sync-hook 解决(那会让 Phase 2.1 超出 installer 范围);verify 用 best-effort filesystem scan,目录冲突作为已知 limitation 记 progress.md finding,真实桥接推后续 phase。

### anthropics/skills license policy（gray area 2）
- **D-03:** license whitelist 加 **`anthropics-official` carve-out** — `frontend-design` / `document-skills` / `webapp-testing` 来自 `anthropics/skills`(无 LICENSE 文件,source-available 非 OSI)。加一个 `anthropics-official` license class 专门放 anthropics/skills 来源。项目已隐性接受(decision_rules.yaml 已含它们)——这是正式化。
- **D-04:** `anthropics-official` carve-out 是架构/policy 决策,**写进 ADR 0010 § Decision**(不是悄悄改 whitelist)。同时加 `license_source` audit 字段(README/registry/none/anthropics-official)记明 license 来源出处。
- **D-05:** baoyu-skills "license None warn"(D1.2.5-10)research 已 resolve 为 **MIT-0** — 加 MIT-0 到 whitelist,去掉 decision_rules.yaml `chinese-content-deck` rule 的 `warn:`(这条 fold 进 Wave 0,因为是 license whitelist 工作的一部分)。

### Wave 0 packaging（gray area 3）
- **D-06:** phase 1.5 sister review deferred + research refinement #8/#10/#11 打包为 **Phase 2.1 Wave 0 独立任务批**,在 4 个 installer 实装 wave 之前跑。progress.md 单独追踪。沿袭 phase 1.5 的 8-wave 模式。
