### D-02 DisclosureLevel: FULL per-task (raw prompt + decisions + miss reasons + recovery path)

- **Decision**: `docs/benchmarks/v0.4.md` per-task disclosure FULL — 每 task 1 section 含: (a) 原始 prompt verbatim / (b) routing decision (tier + skill + rationale) / (c) actual command executed / (d) hit/miss verdict / (e) miss recovery path (if miss; current 30/30 100% → 全部 hit, recovery path = N/A 但 schema 保留 column for v0.5+ miss case)
- **Rationale**:
  - sister Karpathy transparency precedent (R9.2 透明度) — "路由决策 100% 可追溯"
  - 反 ROADMAP L222 "数据美化诱惑" 风险 — 原始 prompt + decision + verdict 公开使任何人可复现 verify (R8.1 验收 literal)
  - HYBRID (FULL+aggregate header) 评估为 +1 section overhead 微 over-engineering — FULL only 已经隐含 aggregate (footer 30/30 + per-tier ratio 一行总结)
  - AGGREGATE-only 违 R8.1 "原始数据公开任何人可复现" hard requirement, 直接 rejected
- **Sneak-block守门**:
  - planner / executor MUST publish raw prompt (NOT sanitized/paraphrased)
  - planner / executor MUST publish actual decision_rules.yaml routing path (rule_id + tier + skill)
  - planner / executor MUST NOT suppress miss cases (current 30/30 但 schema 保留 recovery_path column for transparency)

### D-03 e2eRecording: TEXT LOG plain `.log` file (grep-able / reproducible / zero-dep)

- **Decision**: 上游升级 e2e 录像 = plain `.log` text file (e.g., `docs/benchmarks/v0.4-upgrade-e2e.log`) — text-only diff + console output capture; **NO asciinema cast / NO mp4/gif** (评估 rejected)
- **Rationale**:
  - sister Karpathy zero-dep precedent — asciinema 需 npm devDep
  - text grep-able + cross-OS portable (Win/Mac/Linux 全 cat-able + diff-able)
  - 0 bloat (text < 50KB vs. mp4 1MB+ per cast)
  - git diff friendly (binary mp4/gif 不可 diff; text log diff = code review friendly)
  - reproducibility: text log 可被任何 reader 验证 vs. video 仅 view-only
- **Sneak-block守门**:
  - planner / executor MUST NOT add asciinema npm dep (违 zero-dep)
  - planner / executor MUST NOT commit binary mp4/gif/png files >100KB
  - planner / executor MUST capture: pre-upgrade state + upgrade command + diff output + post-upgrade state (4-section text log)

  - 对 v1.0-RC 节奏: benchmark 公开 = static publish artifact (`docs/benchmarks/v0.4.md`), 不是 live CI dashboard; manual cadence = doc update frequency, 与 release cadence 一致
- **Sneak-block守门**:
