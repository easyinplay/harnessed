### D-01 (Q1) Master orchestrator behavior = Auto gate-route
- **D-01**: Master `/discuss /plan /task /verify` invoke → 并行 gate-eval per sub-stage via `judgments.<sub>.fires` → fire 那些 sub 满足条件
- Sister CLAUDE.md fallback 铁律 "chain-isolation: 链式互不前置" 机器化 (每 sub 独立判断, NOT 串行 dependency)
- User 仍可 invoke `/discuss-strategic` 等 sub 独立运行 (sub 本身就是独立 slash cmd)
- 不 fire 的 sub → 透明声明 (sister fallback.yaml `skip_with_transparency` 铁律 1)

### D-02 (Q2) Namespace policy ADR 0030 = Bare slash cmd
- **D-02**: 沿袭 v2.0 bare slash cmd pattern (`/discuss-strategic` / `/plan-architecture` / `/task-clarify` / `/verify-paranoid`)
- Sister ADR 0009 v1.0.2 LOCK "bare slash command per SKILL.md name field; v2.0+ 若引入 namespace 仲裁需 NEW ADR" → v3.0 选 bare = **NOT 引入 namespace**, ADR 0009 LOCK 不动 + NEW ADR 0030 explicit codify "v3.0 confirmed bare convention"
- 与 gstack convention (bare) 一致; gsd-* / harnessed-mattpocock-* 等 plugin-prefix component 不动 (它们是 upstream, harnessed 是 composition orchestrator)

### D-03 (Q3) workflows/ directory structure = Nested stage-grouped
- **D-03**: `workflows/<stage>/<sub>/workflow.yaml + SKILL.md` (nested 2-level)
- Visual 4-stage 架构清晰 (sister CLAUDE.md cadence verbatim 镜像)

- ADR 0024 workflow schema v2 → v3 schema_version bump (sister Phase 2.4 W0.1 pattern)

### D-04 (Q4) v2.0.1 → v3.0 migration = Pure ship v3 deprecate v2
- **D-04**: v3.0 完全 replace v2 workflows (sister Pure bundled D-01 / D-05 release-notes-only pattern):

  - KEEP `/research` (名字不变 standalone)
  - NEW `/retro` standalone (sister CLAUDE.md milestone close)

### D-05 心法 + 招式 cross-cutting (per user clarification 2026-05-20)
