## 三大研究领域关键发现

### Phase 2.2 — execute-task workflow + ralph-loop(`v0.2.0-execute-task-ralph.md`,HIGH conf)

- **SDK AgentDefinition 真实只有 4 字段**(`description`/`prompt`/`tools?`/`model?`)vs harnessed contract 文档化的 14 字段 —— 真实的 "interface exists ≠ production-ready" gap(phase 1.5 H1 教训复现)。Phase 2.2 必须先对照 installed `.d.ts` 重新验证 14 字段 contract,不在 SDK type 里的字段保持 factory-internal。
- **完成验证有比 `<promise>` grep 更好的模式**:SDK 现有一等 structured output(`outputFormat: { type: 'json_schema' }` → typed `structured_output` field)。**结构性消除** F33 summarize-risk。建议 dual-signal:`structured_output` PRIMARY + `<promise>COMPLETE</promise>` grep FALLBACK。
- execute-task orchestration:每 subtask 一个 `query()`,ralph-loop wrap deliver phase,SDK `resume` 原生跨迭代携带 context(干掉手动 re-injection hack),`SDKResultMessage.subtype` discriminated union 驱动 loop exit。
- **建议 ADR 0010** — 记录 SDK 引入 + ralph-wiggum keep-vs-switch 决策。
- ⚠️ open question:`outputFormat` + `agents`-map 组合未见联合文档 → Phase 2.2 spike-test 风险(先测 1 sample 再跑 30)。

### Phase 2.1 — 4 installer methods(`v0.2.0-installers.md`,HIGH 3/4 + MEDIUM 1/4)

- **cc-plugin-marketplace UNBLOCKED**(最大 refresh 发现)— CC 现有一等非交互 `claude plugin marketplace add` + `claude plugin install <plugin>@<marketplace> --scope project`(2026-05-11 时还是 `/plugin` slash-command only)。完全 headless。
- **mcp-http-add ≈ mcpStdioAdd 的 1:1 clone** — `claude mcp add --transport http --scope project <name> <url>`;CC v2.1.122 user-scope bug(#54803)transport-agnostic,同样 hardcode `--scope project`。**建议最先实装**(proven sibling pattern)。
- **git-clone-with-setup HIGH** — ui-ux-pro-max.yaml 已是真实可用 manifest,full-SHA pinned,过 B1 security gate。win32 需 Git Bash。
- **npx-skill-installer MEDIUM / soft-defer 候选** — 无单一官方 Anthropic 工具,且 `~/.agents/` vs `~/.claude/skills/` 目录约定冲突使 verify 难。建议:minimal pin 一个工具,或部分 deferred(ui-ux-pro-max 风格 skill 可走 git-clone-with-setup)。
- **实装顺序**:mcp-http-add → git-clone-with-setup → cc-plugin-marketplace → npx-skill-installer(低风险优先)。

### Phase 2.3 — design/content/testing extension(`v0.2.0-extensions.md`,HIGH,7/7 候选 repo+license 经 `gh api` 验证)

- **ui-ux-pro-max MOVED** — 不再是 `midwayjs/midway` v4-next branch(phase 1.3 PATH_A+B 目标),现有专属 repo `nextlevelbuilder/ui-ux-pro-max-skill`(MIT,~71k stars)+ 一等 cc-plugin-marketplace 支持。manifest 需 re-point —— clean upgrade。
- **3 候选共享 1 个 license 问题** — `frontend-design` / `document-skills`(pptx/docx/xlsx/pdf)/ `webapp-testing` 全来自 `anthropics/skills`,**无 LICENSE 文件**(source-available 非 OSI),fail harnessed whitelist。单一前置:加 `anthropics-official` license carve-out。
- **D1.2.5-10 RESOLVED** — baoyu-skills "license None warn" → 实为 **MIT-0**(registry-declared)。加 MIT-0 到 whitelist,去掉 `chinese-content-deck` rule 的 `warn:`。
- **document-skills 是 bundled plugin** — 装一次得 4 个 doc skill 作为一个 unit,非 4 个 atomic install → manifest schema 需建模 bundle-install(与 Phase 2.1 ManifestSpec 工作协调)。
- playwright-cli(Apache-2.0,npx-skill-installer)+ chrome-devtools-mcp(Apache-2.0,mcp-stdio-add)干净。
- **7/7 候选全 READY**(无 should-wait)。3 个 license-policy action item 是唯一真实前置。
