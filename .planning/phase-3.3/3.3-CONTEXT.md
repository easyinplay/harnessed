## Implementation Decisions

### D-01: aliases.yaml schema 形态 = RICH (含 reason + since_version + deprecation_date)

**Selected**: RICH — `aliases.<old-name>: { redirect: <new-name>, reason: 'upstream renamed', since_version: '0.3.0', deprecation_date: 'YYYY-MM-DD', removal_date?: 'YYYY-MM-DD' }`

**Locked rationale**: R7.5 验收"deprecation marker 显示提示"含蓄要求 metadata (FLAT map 仅给 redirect 不能解释 why/when). TIERED union 复杂化 (Karpathy YAGNI 违反). RICH 直观结构, sister Phase 2.4 doctor `--json` flag CI-friendly 输出可直接展开 metadata 给 doctor warning.

**Implementation hint**:
- `manifests/aliases.yaml` 顶层 yaml file (sister manifests/ flat dir pattern)
- `src/manifest/schema/aliases.v1.ts` NEW ~30L — TypeBox schema:
  ```ts
  export const AliasEntryV1Schema = Type.Object({
    redirect: Type.String({ minLength: 1 }),
    reason: Type.String({ minLength: 1 }),
    since_version: Type.String({ pattern: '^\\d+\\.\\d+\\.\\d+$' }),
    deprecation_date: Type.String({ pattern: '^\\d{4}-\\d{2}-\\d{2}$' }),  // ISO-date sister Phase 3.1 D-01 strict pattern (NOT format date-time; FormatRegistry not 注册 sister W2 Rule 1 lesson)
    removal_date: Type.Optional(Type.String({ pattern: '^\\d{4}-\\d{2}-\\d{2}$' })),
  }, { additionalProperties: false })
  export const AliasesV1Schema = Type.Object({
    schemaVersion: Type.Literal('harnessed.aliases.v1'),
    aliases: Type.Record(Type.String({ minLength: 1 }), AliasEntryV1Schema),
  }, { additionalProperties: false })
  ```
- `src/manifest/aliases.ts` NEW ~30L — `loadAliases()` + `resolveAlias(name): string | null`
- Phase 1.X+2.X install path 加 redirect step (在 manifest lookup 前 1 line `name = resolveAlias(name) ?? name`)
- Karpathy hard limit: aliases.ts ≤40L; aliases.v1.ts ≤40L

**decision_source**: 用户首轮决策 RICH (Recommended option)

### D-02: deprecation marker 可见性策略 = DOCTOR-ONLY-WARN (install 静默重定向)

**Selected**: DOCTOR-ONLY-WARN — doctor 加 7th check 显示 deprecation warning + install 静默重定向 (无 warning)

**Locked rationale**: Karpathy YAGNI 中庸 — INSTALL-WARN 增 CI 噪声 (同 manifest 多次 install 重复警告 + log clutter); INSTALL-THROW 太严苛 (用户已知装 deprecated 还被 block 不合理 + 需 `--force-deprecated` flag 增 CLI surface). DOCTOR-ONLY 把 deprecation 提醒放健康检查路径 (人主动查) — install path 保持 clean redirect (R7.5 验收"install 通过"语义对齐).

**Implementation hint**:
- `src/cli/doctor.ts` 加 7th check `checkDeprecations` (sister 6th check probe-gstack 模式 dynamic import + delegate):
  ```ts
  async function checkDeprecations(): Promise<CheckResult> {
    const { listDeprecations } = await import('./lib/check-deprecations.js')
    const r = listDeprecations()
    return { name: 'deprecated manifests', status: r.length > 0 ? 'warn' : 'pass', message: ... }
  }
  ```
- `src/cli/lib/check-deprecations.ts` NEW ~30L PRIMARY helper (sister probe-gstack.ts pattern; doctor.ts 186→~193L 仍 ≤200 Karpathy clean)
- install path (Phase 1.X+2.X) 仅加 `name = resolveAlias(name) ?? name` 1 行 (no console output, sister Karpathy surgical)
- doctor warning output 含 RICH metadata: "Manifest 'old-name' deprecated since 0.3.0 (date 2026-05-17), redirects to 'new-name'. Reason: upstream renamed."
- `--json` flag 输出 deprecations array (CI-friendly sister Phase 2.4 D-01)
- Karpathy hard limit: check-deprecations.ts ≤40L

**decision_source**: 用户首轮决策 DOCTOR-ONLY-WARN (Recommended option)
