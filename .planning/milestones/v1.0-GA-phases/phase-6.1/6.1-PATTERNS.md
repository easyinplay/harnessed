```yaml
  publish:
    name: npm publish
    runs-on: ubuntu-latest
    defaults:
      run:
        shell: bash
    steps:
      - uses: actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5 # v4.3.1
      - uses: actions/setup-node@49933ea5288caeca8642d1e84afbd3f7d6820020 # v4.4.0
        with:
          node-version: 22
          registry-url: 'https://registry.npmjs.org'
```

**Build + publish core** (ci.yml lines 150-175 pnpm pattern adapted):
```yaml
      - run: corepack enable
      - run: corepack pnpm install --frozen-lockfile
      - run: corepack pnpm build
      - run: npm publish --provenance --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

**Key delta from ci.yml:**
- Trigger: `tags: v[0-9]+.[0-9]+.[0-9]+` (NOT branches/PR)
- Permissions: add `id-token: write` (OIDC provenance)
- No matrix (ubuntu-latest only; publish is idempotent single-run)
- No A7 check, no test steps — publish-only job
- SHA-pinned action refs copy verbatim from ci.yml (supply-chain hardening per Phase 1.1.1 H1)
- `registry-url` on setup-node required for `NODE_AUTH_TOKEN` env resolution
- `npm publish --provenance --access public` (D-02 LOCKED; NOT `pnpm publish`)
