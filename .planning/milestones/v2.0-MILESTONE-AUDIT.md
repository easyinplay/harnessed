## Architectural delivery summary

### Pure bundled SoT distribution model
- `<packageRoot>/workflows/` readonly + share-only (NO user override)
- 升级 cadence: maintainer 改 packageRoot + ADR 0024+ + patch release 2-3 day cycle (NOT 1-2 week major cycle)
- End-user 体验: `npm install -g harnessed@2.0 && harnessed setup --apply` 一行 immediate upgrade

### 4 workflows 完整 4-stage 三层栈机器化
- **Stage ① Discuss**: `research` workflow (Tavily + Exa + ctx7 多源 fan-out + GSD synth)

- **Stage ③ Execute**: `execute-task` workflow v2 (brainstorming + karpathy + mattpocock route + TDD + ralph-loop COMPLETE)

## v2.0 lessons learned

1. **Q-AUDIT post-LOCK audit value**: 3 P1 finding caught after Phase 2.1 LOCK (planning-with-files terminology drift + Agent Teams schema + judgmentResolver gap) — post-LOCK audit (Phase 2.2 Wave A research + gstack /plan-eng-review) catches what initial discuss misses。
2. **Dogfood-first methodology proven**: Schema-shape regex tests STATIC pass while runtime evalGate FAIL — only real fs.readFile + parseYaml + evalGate dogfood catches production bug。
3. **Agent Teams cost vs benefit**: Pattern A 全栈三路 ~3× subagent fan-out token cost; justified ONLY for API contract cross-deps (Phase 2.4 W1 3-teammate)。File-independent tasks (Phase 2.4 W2 + W3) subagent fan-out 更优。
4. **PLAN.md file-path drift**: T2.4.W1.2 PLAN.md L340 wrote `src/workflow/engineHook.ts` but actual catch site `src/routing/engine.ts:L188-199` — planner-time file-location can stale before execution-time。Mitigation: Wave 0 schema task should include file-location verify step (deferred-items.md lessons learned)。
5. **expr-eval 2.0.2 grammar case-sensitivity**: RESEARCH § 1.2 documented `AND`/`OR` uppercase 等价 lowercase, 实测 only lowercase accepts。Caught Phase 2.5 W2 dogfood-first (sister ADR 0026 + 0028 errata)。
