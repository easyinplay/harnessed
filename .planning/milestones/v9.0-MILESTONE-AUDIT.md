## Milestone Discuss Decisions (carried forward 2026-06-24)

Preserved here at close — the ROADMAP active block collapses to a one-line shipped-index row, so these are the durable rationale:

1. **Scope B (locked via `/plan-eng-review`)** — descriptor + detect + central resolvers + ONE real second-platform proof. Rejected scope C (5-platform matrix, over-build — upstreams already portable) and scope A (seam-only, YAGNI — one implementation never validates an abstraction).
2. **Second-platform pivot `.agents/` → Codex (D-28a)** — anti-stale verification (2 research subagents, host + official docs) found `~/.agents/` is a skills-ONLY convention (vercel-labs/skills; canonical store that `.claude` symlinks into), NOT a harness home. Cursor rejected (uninstalled + opaque app-settings DB; only `~/.cursor/mcp.json` filesystem-addressable). Codex is the only host-verifiable real full harness.
3. **Codex write surfaces = capability-absent (D-28b)** — Codex settings+MCP live in one `config.toml` (TOML); harnessed's JSON env-key writer doesn't port, and the CC env keys (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS`) are CC-specific. No TOML writer; the descriptor's `supportsEnvKeyWrite=false` gates them off. The seam's tolerance for a divergent partial-capability platform IS the validation.
4. **claude-first precedence = zero blast** — every existing user gets byte-identical behavior; a second platform is reached ONLY via explicit opt-in (`HARNESSED_PLATFORM` / `setup --platform` / `.platform` pin) or a non-claude host.
