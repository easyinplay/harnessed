## Summarize — checkpoint 摘要生成策略

| Option | Description | Selected |
|--------|-------------|----------|
| TEMPLATE | 机械 extract 固定 fields (phase_num / status / last_task / key_decisions). Zero LLM call / deterministic / 极快 / 可纯代码验证. Karpathy "最小有效". R7.2 验收 < 1k token 明显指向机械. | ✓ |
| LLM | 每次 checkpoint 调 agent 做语义压缩 (1 SDK call per checkpoint). 表达力强可保留 nuance. 劣: API 依赖 + cost + 不 deterministic + 可能超 1k 需 retry. | |
| HYBRID | template default + LLM optional 触发. 灵活但 2 条路径双 schema 加 wire 复杂度. | |

**User's choice:** TEMPLATE (Recommended option)
**Notes:** zero LLM call + deterministic 是 infra 底座必备品; LLM 是 over-engineering. R7.2 验收"< 1k token"明指机械.

## StateModel — current-workflow.json 状态机模型

| Option | Description | Selected |
|--------|-------------|----------|
| KARPATHY | `active` / `paused` / `complete` 3 state. YAGNI 不开 FSM. resume 逻辑 = `if status==='paused' load checkpoint`. | ✓ |
| STRING | 极端简化 `phase_X_in_progress` / `phase_X_complete` 单 string. paused 状态难表达 (需额外 metadata). | |
| FSM | 6 state finite-state-machine (idle/discussing/planning/executing/awaiting-checkpoint/recovering). 教科书完整但 phase 3.1 MVP 不需也不能测全部转移. | |

**User's choice:** KARPATHY (Recommended option)
**Notes:** 3 state 足够表达"在跑/中断了/跑完了"实际语义. 字符串 enum 比 nested object 简单 50%. xstate / robot3 等 lib 不引入.

## Resume — harnessed resume 走什么行为

| Option | Description | Selected |
|--------|-------------|----------|
| RELOAD | `harnessed resume` 输出 checkpoint 摘要 + 提示用户手动在 Claude Code 内调起 phase command 续跑. 不偷袭用户. | ✓ |
| AUTO-REPLAY | resume 自动调起 last phase command 续跑. 一行完事但需 reliable command-rehydration (Claude Code 环境外不一定能调起 slash command). | |

**User's choice:** RELOAD (Recommended option)
**Notes:** Claude Code 交互底色 = 用户总需手动 invoke skill. RELOAD 多报一行命令 << AUTO-REPLAY 黑盒决策路径风险.

## T4.4 Wire — T4.4 Task Session closure infra 是否 wire-in checkpoint flow 同 ship

| Option | Description | Selected |
|--------|-------------|----------|
| WIRE-IN | session_id 写 checkpoint, resume 时通过 `ralphLoopWrap.resumeSessionId(sid)` SDK redirect 续原 session. SDK session 续跑 = lossless conversation context 恢复. | ✓ |
| DEFER | T4.4 closure infra 保留 unused, fresh session each resume + reload checkpoint 摘要重建 context. 依赖面窄但损 conversation context. | |

**User's choice:** WIRE-IN (Recommended option)
**Notes:** Phase 2.2 T4.4 三件套 closure infra 已 ready 1 milestone, defer 又 1 milestone 就成 dead infra. 实现成本极低 (~20-30L), 业务价值 (lossless session 续跑) 远高于 reload 80% 摘要损耗.
