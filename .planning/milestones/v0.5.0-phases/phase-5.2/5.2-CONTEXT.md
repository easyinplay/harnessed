### R10.4 path traversal regex hardening

#### D-03 AttackVectorScope: 5 vector minimal MVP (HIGH CSO, deliberate)
- **Source**: Q3 user deliberate; gstack briefing CSO + Paranoid Staff Eng 视角
- **5 patterns** (OWASP A1 path traversal top 5):
  1. Unix dot-dot: `\.\.\/` regex catch `../etc/passwd` / `../../sensitive`
  2. Windows back-dot: `\.\.\\` regex catch `..\..\windows\system32`
  3. Null byte injection: `\x00` regex catch `legit/path\0attack/path`
  4. URL-encoded dot-dot: `%2[eE]%2[eE]` regex catch `%2e%2e%2fetc%2fpasswd`
  5. Double-encoded: `%25[2][eE]%25[2][eE]` regex catch `%252e%252e%252f` (defense against decode-then-validate flow)
- **Sneak-block**:
  - NO comprehensive 10+ vectors (symlink / UNC \\\\server\\share / Unicode NFD/NFC / Win short name 8.3 / NTFS ADS — 超 Karpathy YAGNI; v0.6+ if real attack signal)
  - NO regex 黑名单 + path.resolve() 白名单双防 (defense-in-depth 优秀 但需 fs.realpath + Win symlink edge case + 2x test surface; defer v0.6+ if signal)
  - NO 仅加 1 vector (违 R10.4 spec verbatim "5+ vectors" 不可接受)
- **Implementation**: pre-compile RegExp instances at module load (sister Phase 5.1 R10.1 redact 5 patterns 同 pattern); throw `PathTraversalError extends Error` with safe message (NOT leak attempted path per Q8 D-08)
- **Rationale**: R10.4 spec verbatim minimum + Karpathy YAGNI + sister Phase 5.1 redact pattern symmetric design

#### D-04 HardeningScope: resolveAlias + manifest path 2 点 minimal (HIGH, deliberate)
- **Source**: Q4 user deliberate; gstack briefing Paranoid Staff Eng + CSO surface area minimal
- **2 sites** (user-controlled path inputs only):
  1. `src/manifest/aliases.ts` `resolveAlias()` — user passes `<name>` to harnessed install/uninstall/audit-log → aliases.yaml lookup → potential traversal via crafted alias entry
  2. `src/cli/install.ts` L82 + `src/cli/uninstall.ts` NEW similar — `resolve(process.cwd(), 'manifests/tools/${name}.yaml')` user-controlled `name` parameter
- **Sneak-block**:
  - NO src/audit/log.ts hardcoded `.harnessed/audit.log` path (NOT user-controlled, no attack surface)
  - NO uninstall.ts redundant explicit verify (transitive resolveAlias 保护 already covers)
  - NO comprehensive 全 fs.readFile/writeFile/spawn args audit (超 Phase 5.2 1-day target; sister BB path 4 项 carry-forward NOT v1.0 GA gate)
- **Rationale**: minimal attack surface targeting real user-input paths;Karpathy YAGNI

#### D-06 ConfirmPrompt: --yes bypass interactive y/N (MED, batch)
- **Source**: Q6 batch confirm 按推荐
- **Interactive default**: prompt `Confirm uninstall '<name>'? y/N` 默认 No
- **CI/script**: `--yes` flag skip prompt + 必须 --apply (sister install --non-interactive --apply double-flag pattern延袭)
- **Rationale**: 双重保护 user shooting foot;CI 自动化 path 不阻塞

#### D-07 BackupOption: NO --keep-backup flag (LOW, batch)
- **Source**: Q7 batch confirm 按推荐
- **Decision**: 不提供 `--keep-backup` flag; delete 直接 NOT move-to-backup
- **Alternative**: 已有 `harnessed backup` 独立 workflow (Phase 2.1+ ship sister installer backup pattern); 用户需 backup 先跑 `harnessed backup create` 后再 uninstall
- **Rationale**: single responsibility;backup 是 backup 子命令的职责 NOT uninstall 子命令

#### D-08 ErrorMessageSafety: NOT leak attempted path (LOW CSO, batch)
- **Source**: Q8 batch confirm 按推荐
- **Pattern**: PathTraversalError message generic `"path traversal attempt detected"` NOT include user input attempted path (例如 NOT echo back `../../etc/passwd` reveal attacker enumeration success)
- **Logging side**: structured log internal (audit.log Phase 4.3 producer 可补 NEW audit category `security_violation`) but user-facing error generic
- **Rationale**: defense in depth — error message itself NOT becomes attack reconnaissance channel; sister CSO Paranoid Staff Eng veto
