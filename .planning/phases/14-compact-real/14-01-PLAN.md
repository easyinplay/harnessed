2. **G6 regression（F1 fix）**：一个 `{status:'done', fail_count:3}` 条目**不被驱逐**（fail_count>0 必保）；且把 compactLedger 的 `kept` 喂给 `breakLoop` 仍返回该 sub（`import { detectLoop } from './breakLoop.js'`，断言命中）。

5. token 度量：返回 before/after token（`Buffer/4`），`pct_saved` 为 token 百分比（非条目数）。
6. 空 ledger → `{ kept:[], evicted:0, pct_saved:0, ... }` no-throw。
当前无 `compactLedger` → 全 red。
  </red>
  <green>
实现纯 `compactLedger(entries: SubProgressEntryType[])`：
- `const evictable = e => (e.status==='done'||e.status==='skipped'||e.status==='rejected') && (e.fail_count ?? 0) === 0`
- `kept = entries.filter(e => !evictable(e))`；`evictedEntries = entries.filter(evictable)`。
- `by_status` = evicted 按 status 计数；`evicted = evictedEntries.length`。
- token：`tok = x => Math.ceil(Buffer.byteLength(JSON.stringify(x))/4)`；`before=tok(entries)`、`after=tok(kept)`；`pct_saved = before>0 ? Math.round((before-after)/before*100) : 0`。
- `digest` = 一行 string，如 `compacted N entries (done:a skipped:b rejected:c)`。
- 返回 `{ kept, evicted, by_status, before_tokens:before, after_tokens:after, pct_saved, digest }`。纯函数零 I/O。
