## 8 D-decisions LOCKED (4 HIGH deliberate + 4 MED/LOW batch confirm)

### R10.1 audit log --filter CLI consumer

#### D-01 FilterSyntax: jq pipe 内嵌 invocation (HIGH, deliberate)
- **Source**: Q1 user deliberate; gstack briefing CEO/Eng/Designer/Paranoid Staff Eng 视角对齐 jq Unix 哲学 + zero-new-dep
- **CLI UX**: `harnessed audit log --filter '.category=="engineering"' --tail 50` → 内部 spawn `jq` subprocess pipe `.harnessed/audit.log` 内容 进 jq stdin, 输出回 CLI stdout
- **Dep status**: jq runtime 已存在 (doctor 3rd check verify; Phase 1.2 install_method `mcp-stdio-add` 隐式依赖); zero-new-dep
- **Sneak-block**:
  - NO JSONPath npm dep (jsonpath-plus ~30KB 维护负担 + Python familiar non-fit harnessed TS/JS 受众)
  - NO 自写 DSL parser (表达力弱 OR/AND/range 缺 + Karpathy YAGNI)
  - NO `harnessed audit log | jq` raw pipe (UX 差 + Win CMD 无 jq 时降级失败)
- **Implementation**: `child_process.spawn('jq', [filterExpr], { stdio: ['pipe', 'inherit', 'inherit'] })` + appendFileSync read audit.log → write jq stdin
- **Rationale**: jq Unix 哲学 + 表达力极强 + 工程师 familiar + doctor 已 verify

#### D-02 OutputFormat: dual format default human table + --json flag opt-in (MED, batch confirm)
- **Source**: Q2 batch confirm 按推荐
- **Default**: human-readable table 5 关键列 `ts | phase | category | matched_rule_id | outcome` (12-field condense)
- **Opt-in**: `--json` flag 完整 12 字段 JSON pretty-print pipe-friendly
- **Implementation**: `--json` flag → `JSON.stringify(record, null, 2)`; default → manual column align (cli-table3 NOT used per Karpathy YAGNI minimal dep)
- **Rationale**: human-first + pipe-friendly opt-in;sister industry pattern (kubectl get / docker ps)

#### D-03 PaginationFlags: minimal MVP 3 flag (LOW, batch confirm)
- **Source**: Q3 batch confirm 按推荐
- **Flags** (Phase 5.1 ship 3 个):
  - `--tail N` recent N records (default 50)
  - `--head N` oldest N records
  - `--reverse` flip output order
- **Deferred v0.6+ if signal**: `--since <ISO>` + `--until <ISO>` + `--sort <field>` (Karpathy YAGNI no current signal)
- **Rationale**: Karpathy YAGNI minimal MVP + Unix tail/head familiar + reverse 兜底排序需求

#### D-04 SecurityRedact: consumer 2重 redact 5 patterns (HIGH CSO, deliberate)
- **Source**: Q4 user deliberate; gstack briefing CSO Paranoid Staff Eng 视角defense-in-depth
- **Patterns** (consumer 输出前 regex match + replace `[REDACTED]`):
  1. `api_key=<value>` / `api_key:\s*<value>` (anywhere in task_excerpt)
  2. `token=<value>` / `token:\s*<value>`
  3. `password=<value>` / `password:\s*<value>`
  4. `Authorization: Bearer <token>` (HTTP header literal)
  5. Key prefixes: `sk-` / `pk-` / `gh_` / `ghp_` / `ya29.` / `AIza` (catch GH/OpenAI/Google token leak pre-`=`)
- **Defense-in-depth**: producer (Phase 4.3 `src/audit/log.ts`) 已 sanitize (verify needed in execute-phase research wave) + consumer 二重 redact (即使 producer coverage 不全 also 防);双层保险
- **Producer audit deferred verify**: Phase 5.1 W0 task scope (NOT W2 ship blocker; producer 已 ship verified Phase 4.3 R8.1 acceptance,W0 verify-only NOT FIX-FORWARD)
- **Sneak-block**:
  - NO 仅依赖 producer (R8.1 spec 未 verbatim 要求 sanitize → producer coverage 可能不全)
  - NO defer redact 到 Phase 5.2 (audit log = security boundary 不能延迟 CSO Paranoid Staff Eng 不批准)
- **Rationale**: defense-in-depth + CSO Paranoid Staff Eng veto-or-accept gate

### R10.2 state.ts concurrent write lock (#BU)

#### D-05 LockImpl: proper-lockfile npm dep (HIGH, deliberate)
- **Source**: Q5 user deliberate; gstack briefing Paranoid Staff Eng + EM 视角对齐 battle-tested dep
- **Library**: `proper-lockfile` (5M weekly downloads, MIT, 6 KB minified, no runtime dep, cross-OS)
- **API**: `lockfile.lock('.harnessed', { stale: 10000, retries: { retries: 3, factor: 2, minTimeout: 100 } })` + `release()` in finally
- **Cross-OS**: Win ACL workaround 已知 (sister CLAUDE.md corepack ACL 教训共享思路;proper-lockfile 自带 Win fallback)
- **Sneak-block**:
  - NO 手写 `fs.open(wx)` exclusive create + retry (race window minimal but exists + cleanup process crash leftover lock file 手处理 footgun)
  - NO OS flock (Win 不支持 — cross-OS Day 1 ci.yml gate 强制必须 cross-OS)
  - NO defer Phase 5.2+ (sister 4th-cycle BB path 已 lock 4 项 carry-forward; Phase 5.1 必须兑现 R10.2 推 v1.0 GA)
- **Rationale**: dep 1 个换 robust + cross-OS;sister Win corepack ACL workaround 教训 — 手写 cross-OS file lock 是 footgun

#### D-06 RetryPolicy: bounded retry 3×100ms exp backoff + clear error (MED, batch confirm)
- **Source**: Q6 batch confirm 按推荐
- **Policy**: `{ retries: 3, factor: 2, minTimeout: 100 }` → ~700ms total (100 + 200 + 400) → throw `LockHeldError` with hint
- **Error message**: `"another harnessed process holds the lock at .harnessed/.lock — wait or kill stale process (try: harnessed status)"`
- **Sneak-block**:
  - NO unbounded retry wait forever (死锁风险 leftover lock)
  - NO fail-fast NO retry (UX 弱 transient lock window 立即 fail)
- **Rationale**: bounded 死锁 mitigate + UX 友好 + 错误 actionable

#### D-07 ForceFlag: NO --force flag + harnessed status 显示 lock holder (LOW, batch confirm)
- **Source**: Q7 batch confirm 按推荐
- **Decision**: 不提供 `harnessed resume --force` bypass lock
- **Alternative UX**: `harnessed status` 显示 `.harnessed/.lock` holder pid + timestamp; stale 判定靠 `proper-lockfile` 自带 10s 超时
- **Sneak-block**:
  - NO --force flag (race condition 可能 lock 被真活着的 process 持有 — --force 会损坏 state)
  - NO 用户必须 kill stale process (--force 缺失 + status 显示 + stale auto-detect = 完整 UX 但无危险)
- **Rationale**: 防 user shooting foot + proper-lockfile stale 自动检测充分

#### D-08 LockScope: dir-level lock `.harnessed/.lock` (HIGH, deliberate)
- **Source**: Q8 user deliberate; gstack briefing CSO + Paranoid Staff Eng 视角 minimal-yet-complete
- **Scope**: 整个 `.harnessed/` dir lock by `proper-lockfile`
- **Protects**: writeCurrentWorkflow + engineHook activatePhase/completePhase + readCurrentWorkflow + readCheckpoint + writeCheckpoint + archive (所有 .harnessed/* 写)
- **Granularity**: 粗 (1 lock for all writes) but simple + 安全
- **Sneak-block**:
  - NO 单文件 lock current-workflow.json (engineHook write 其他 checkpoint files 仍 race)
  - NO operation-level lock 7 fn 各锁自己 (Karpathy YAGNI 5x 复杂度)
  - NO process-level lock (harnessed status + resume 不能并行 UX 太强)
- **Refine deferred v0.6+ if signal**: op-level lock if NEW concurrent read pattern triggers contention measurement
- **Rationale**: Phase 5.1 NEW infra YAGNI;粗 granularity = simple + 安全 + sufficient for current dogfood + v0.6+ refine path open
