## Trusted Publishers UI — Deferred Flow

**Why deferred:** UI config requires the `harnessed` package to exist on npm first
(Trusted Publishers is configurable post-first-publish). The workflow code (T1.1)
can be written and committed now; the UI config is completed at tag-push time (Wave 2 T2.10).

**Fallback option (if Trusted Publishers not configured before tag push):**
The `publish.yml` includes `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` env reference.
If OIDC Trusted Publisher config is not complete before the v1.0.0 tag push:
1. Create a granular npm access token at npmjs.com → Access Tokens → Generate New Token
2. Add as GitHub repository secret: `Settings → Secrets → NPM_TOKEN`
3. The workflow `env: NODE_AUTH_TOKEN` will pick it up automatically

**Preferred path (no fallback needed if UI config done before tag push):**
1. After T1.1 is merged to main, navigate to:
   `https://www.npmjs.com/package/harnessed/access` (post-first-publish)
   OR: npmjs.com → Profile → Packages → harnessed → Settings → Trusted Publishers
2. Add publisher: owner=`easyinplay`, repo=`harnessed`, workflow=`publish.yml`
3. No secret needed in GitHub repo settings — OIDC tokenless
