### D-03: EE-5 5-question merge gate 安装位置 = BOTH

**Selected**: BOTH(CLI prompt warn-only + plan-phase template hard reject)

**Locked rationale**: 双层防 thin wrapper 反 import 别人产品身份 — CLI 层 `harnessed manifest add <upstream>` 拦截轻量 path(开发者 ad-hoc 加 adapter), plan-phase 层 hard reject 拦截重量 path(正式 phase plan 引入新装配)。 单层都不够:CLI-only 可被 plan-phase 绕过;plan-phase-only 错失 ad-hoc 触发点。

**Implementation hint**: CLI 互动 5 问 prompt(intel L86 5 题原型:真 reusable surface / fit shape / overlap / import 概念 vs 身份 / user 理解), `--non-interactive` 跳 dry-run-only(不绕过 hard reject — dry-run 只是不动 manifest state)。 plan-phase 模板加 5 题清单节, gsd-plan-checker BLOCKER if 5 问 未答 (~30L code 双层合)。

**decision_source**: 用户首轮决策 BOTH(2-round 第 1 轮)

### D-04: CD-3 negative-space syntax + 位置 = DR-only(decision_rules SSOT)

**Selected**: DR(`decision_rules.yaml` 加 optional fields `do_not_use_when:` + `if_rejected_use: <component-id>`)
