## CD-3 negative-space syntax + 位置（gray area 4 / D-04）

| Option | Description | Selected |
|--------|-------------|----------|
| DR — decision_rules.yaml 加 optional fields do_not_use_when + if_rejected_use:<id> | 单一 SSOT;arbitrate 一致 consume;~15L code 增量 | ✓ |
| MANIFEST — 每 adapter manifest yaml 加 do_not_use_when 字段 | 装配层声明负空间, 但 schema 分裂(decision_rules + manifest 双源) | |
| HYBRID-BOTH — decision_rules 加规则 + manifest 加自声明 | 两源都改, consumer 路径不一致 | |

**User's choice:** DR — `decision_rules.yaml` 加 optional fields `do_not_use_when:` + `if_rejected_use: <component-id>`

**Notes:** 单一 SSOT — schema 不分裂, 不开第二来源(避免 decision_rules / manifest 双源不一致)。 routing engine `arbitrate()` 一致 consume:读到 negative-space 则 down-rank + reject 时返回 redirect target(~15L code)。 TypeBox optional fields(每 rule entry 可选), `arbitrate()` 在 evaluate matchesTrigger 后检查 `do_not_use_when:` regex/condition 命中 → score -= penalty 或直接 disqualify, reject 时 return `{ rejected: true, redirectTo: rule.if_rejected_use }`。 schema errata 走 ADR(编号 plan-phase 实占, **不预占 0012**)。 alternative-considered: MANIFEST 把负空间分散到各 adapter 自声明, 维护痛(N×M 一致性), HYBRID-BOTH 两源都改 consumer 路径不一致 explicitly schema 分裂。
