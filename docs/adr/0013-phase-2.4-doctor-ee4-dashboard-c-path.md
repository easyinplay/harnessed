### 5. SSE 替 WebSocket rationale (B-23 / D2.4-9)

dashboard C 路径 § 3.2 selection SSE over WebSocket 三层 rationale: **(a) zero-dep** 沿袭 dashboard.mjs Node-stdlib-only 哲学,无需 ws/socket.io 依赖 (`text/event-stream` HTTP header 内建,EventSource 浏览器原生);**(b) 单向 push 语义** STATE.md → frontend update 是 server-push 模式,WebSocket bidirectional 是 over-engineering (YAGNI);**(c) native auto-reconnect** EventSource 失连续传 lastEventId 替手写 ws reconnect helper + heartbeat ping。production corner case (proxy buffering / Cloudflare timeout) 在 dashboard-handoff 文档已 cite mitigation — `X-Accel-Buffering: no` header + 30s heartbeat keepalive。
